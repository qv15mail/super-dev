# Super Dev v2.3.1

发布日期：2026-04-03

## 本次更新重点

- Codex 深度适配升级为 `AGENTS.md + Skills + repo plugin 增强` 的双层模型。
- Codex App/Desktop、Codex CLI、自然语言回退入口现在统一进入同一条 Super Dev 完整流程。
- Claude Code 深度适配升级为 `CLAUDE.md + .claude/skills + ~/.claude/skills + optional plugin enhancement`，不再停留在旧的 commands-first 兼容模型。
- 安装引导、Onboard、Doctor、Detect、Validate 统一改成当前真实宿主入口模型，不再用过期的“slash / 非 slash”二分法误导 Codex。
- 工作流状态机继续收口，`resume / next / continue / doctor / validate / Web API` 统一共享动作卡、恢复语义和流转契约。
- UI 契约治理继续强化：emoji 禁用、runtime 对齐、quality gate、release readiness、proof-pack 持续打通。
- 官网首页、更新历史、双语 README、发布文档、安装文档、版本真源统一升级到 `2.3.1`。

## Codex 适配升级

- 基础主面：项目 `AGENTS.md`、项目 `.agents/skills/super-dev`、`CODEX_HOME/AGENTS.md`、用户级 `~/.agents/skills/super-dev`
- 增强面：repo plugin marketplace 与 `plugins/super-dev-codex/.codex-plugin/plugin.json`
- 推荐入口：
  - Codex App/Desktop：`/super-dev`
  - Codex CLI：`$super-dev`
  - 回退入口：`super-dev: 你的需求`

## 安装与 Onboard 界面修正

- 安装引导不再简单写成“slash 宿主 / 非 slash 宿主”。
- Codex 现在会被明确标记为 skill-first 模式：
  - App/Desktop 用 `/super-dev`
  - CLI 用 `$super-dev`
- Claude Code 现在会被明确标记为 `CLAUDE.md + Skills` 主模型，`commands / agents` 仅作为兼容层保留。
- Onboard 完成页删除了过期的 `/super-dev init` 提示，改成基于当前宿主入口模型的真实指导。
