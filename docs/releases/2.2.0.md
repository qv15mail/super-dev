# Super Dev v2.2.0

发布日期：2026-03-29

## 重点更新

- 重构工作流状态机与恢复链，补齐 `resume / next / SESSION_BRIEF / workflow-state` 语义，支持下班后、宿主关闭后、第二天回来继续当前流程。
- 宿主接入与诊断链路升级，统一 `start / detect / doctor / Web API` 的决策卡与恢复卡，支持更清晰的推荐理由、第一步动作和多宿主分流。
- UI 系统正式接入主流程，新增 `ui-contract.json`、`design-tokens.css`、`ui-contract-alignment`，从需求、文档、实现、runtime、quality 到 release 全链路治理。
- Release / proof-pack / quality gate / release readiness 进一步打通，治理快照、运行时验证、UI 契约执行、交付证据口径统一。

## 宿主与交互

- 正式产品口径统一为 `20` 个统一接入宿主，`OpenClaw` 改为手动插件安装路径。
- 宿主安装、检测、恢复、继续、返工、发布动作语义统一。
- 显式指定宿主时，系统会围绕该宿主给出决策，不再被自动检测结果带偏。
- 加强 Windows / 自定义安装路径发现逻辑，支持 `SUPER_DEV_HOST_PATH_<HOST>` 覆盖。

## UI / 质量治理

- UI 组件生态改为“偏向而非强制”，Web/React 场景明显偏向 `shadcn/ui + Radix + Tailwind`，但允许基于场景选择更合适方案并持续沿用。
- UI review 新增对主题入口、导航骨架、组件导入路径、反模式命中的结构化检查。
- `quality gate`、`proof-pack`、`release readiness`、`frontend-runtime` 均已纳入 UI 契约执行校验。
