# Super Dev v2.3.0

发布日期：2026-03-31

## 重点更新

- 新增 Enforcement 执行层，自动为宿主配置 hooks 和验证脚本，实现编码阶段治理从"建议"升级为"执行"。
- 新增 Memory 记忆系统，支持 4 类记忆（user/feedback/project/reference）+ Dream 整合器，跨会话保留项目知识。
- 新增代码生成器矩阵：Next.js 脚手架、组件脚手架、TypeScript 类型、Tailwind 配置，一键从文档到代码。
- Expert 专家系统 Markdown 化（12 位专家），支持用户自定义专家，新增对抗性验证专家。
- Prompt 生成器重构为分层注册制，9 段优先级架构，支持数据驱动规则和行为约束模板。
- Skill 模板引擎升级，编码前门禁、错误速查、阶段宣告、持续治理通道全面覆盖。
- Pipeline 引擎集成 Hook 系统（8 事件）、上下文压缩、记忆提取、Session Brief 全链路增强。

## 新功能

### Enforcement 系统（治理执行层）

- `super-dev enforce install` — 自动为宿主配置 hooks（PreToolUse emoji 检查等）。
- `super-dev enforce validate` — 运行验证脚本，检查 emoji/import/color/route 合规性。
- `super-dev enforce status` — 查看当前执行层配置状态。
- `super-dev detect --auto` 时自动安装 enforcement hooks。

### Memory 系统（记忆系统）

- `super-dev memory list` — 列出所有项目记忆。
- `super-dev memory show <name>` — 查看指定记忆内容。
- `super-dev memory forget <name>` — 删除指定记忆。
- `super-dev memory consolidate` — 触发 Dream 整合。
- 4 种记忆类型：user、feedback、project、reference。
- MEMORY.md 索引，200 行 / 25KB 自动限制。
- Dream 整合器：4 阶段后台记忆合并（去重、聚合、摘要、写回）。

### 代码生成器

- `super-dev generate scaffold --frontend next` — Next.js App Router 项目脚手架（16 个文件）。
- `super-dev generate components` — UI 组件脚手架（Button/Card/Input/Modal/Nav/Layout）。
- `super-dev generate types` — 从架构文档生成共享 TypeScript 类型。
- `super-dev generate tailwind` — 从 UIUX 设计 tokens 生成 Tailwind 配置。

### Expert 专家系统

- `super-dev experts list` — 列出所有专家（内置 + 自定义）。
- `super-dev experts show <name>` — 查看专家定义。
- 12 位内置专家 Markdown 定义：PM、ARCHITECT、UI、UX、SECURITY、CODE、DBA、QA、DEVOPS、RCA、PRODUCT、VERIFICATION。
- 用户可通过 `.super-dev/experts/*.md` 自定义专家。
- 新增对抗性验证专家（VERIFICATION.md），在质量门禁中担任"红方"角色。

### Hook 系统（钩子系统）

- `super-dev hooks list` — 列出已配置的 hooks。
- `super-dev hooks test <event>` — 测试 hook 执行。
- 8 种 hook 事件：PrePhase、PostPhase、PreDocument、PostDocument、PreQualityGate、PostQualityGate、OnError、SessionStart。
- 在 `super-dev.yaml` 中通过 YAML 配置，支持 Shell 和 Python 执行器。

### Context Compact（上下文压缩）

- `super-dev compact list` — 列出各阶段的压缩摘要。
- `super-dev compact show` — 查看指定阶段的压缩内容。
- 9 段结构化摘要模板，自动在阶段切换时保存/恢复上下文。

### Session Brief（会话状态）

- 10 段标准格式，完整记录当前会话进度、决策、阻塞项。
- Pipeline 完成时自动更新。

## 改进

### Skill 系统

- 全新 Skill 模板引擎，支持宿主特定 frontmatter 渲染。
- Claude Code 兼容 frontmatter：when-to-use、allowed-tools、effort、version。
- 编码前门禁：7 步强制检查清单（技术栈预研、图标库、组件库、API 契约、设计 token、路由、数据模型）。
- 常见错误速查：7 组 wrong-vs-right 代码示例。
- 阶段执行约束：编码期间每个文件自检。
- CLI 命令调用指南：每个 pipeline 阶段的精确命令时机。
- 阶段宣告机制，确保宿主可见当前阶段。
- Research 双引擎文档化（宿主搜索 + 本地知识库）。

### Prompt 生成器

- 分层注册制架构（9 段优先级，高优先级段始终排前）。
- Critical reminders 最高优先级段（永远第一个输出）。
- 技术栈规则：通用预研机制（适用任何框架，不限于 React/Next.js）。
- 行为 Prompt 库：代码风格、虚假声明防御、数字锚定等 6 套约束模板。
- 交叉审查协议：每位专家必须摘要前一位专家的输出。
- 数据驱动规则：从 `knowledge/` 文件加载，不再硬编码。

### 质量与审查

- 4-Agent 并行审查框架（复用 + 质量 + 效率 + 安全）。
- 对抗性验证 Prompt，在质量门禁中引入红方视角。
- Red team 对抗审查 Prompt 生成。
- 审查 Agent 集成到质量门禁流程。

### Pipeline 引擎

- Hook 集成：每个阶段自动触发 PrePhase/PostPhase。
- 上下文压缩：阶段间自动保存/恢复摘要。
- 记忆提取：每个阶段完成后自动提取关键信息。
- Dream 整合：Pipeline 结束时触发记忆合并。
- Session Brief：完成时自动更新会话状态。
- 品牌化阶段输出，带进度指示器（纯 ASCII，全平台兼容）。

### CLAUDE.md 增强

- 编码约束段（在所有编码阶段持续生效）。
- 技术栈预研要求。
- 图标与视觉规则执行。
- 前后端对齐规则。
- 每文件自检要求。
- 阶段宣告要求。
- CLI 命令时机指南。

### UX 改进

- 无参数运行时显示欢迎信息（无项目上下文）。
- `super-dev status` 在空项目上显示"未开始"而非 failed。
- `super-dev setup claude-code` 顶层别名。
- `super-dev init` 自动检测并安装到宿主。
- `--help` 分组显示（核心 / 治理 / 分析）。
- `detect` 与 `doctor` 检测逻辑统一，不再出现矛盾结果。
- 品牌输出使用纯 ASCII 字符，兼容所有终端。
- 版本号全面统一为 2.3.0。

## Web API

- `GET /api/memory` — 列出记忆。
- `DELETE /api/memory` — 删除记忆。
- `POST /api/memory/consolidate` — 触发 Dream 整合。
- `GET /api/hooks` — 列出 hooks。
- `POST /api/hooks/test` — 测试 hook 执行。
- `GET /api/experts/{name}` — 获取专家定义。
- `GET /api/compact` — 列出压缩摘要。
- `GET /api/compact/{phase}` — 获取指定阶段压缩。
- `GET /api/session-brief` — 获取会话状态。
- `POST /api/session-brief/generate` — 生成会话状态模板。
- 共 11 个新端点。

## 后续增强

### Pipeline 智能化

- **Pipeline 状态文件**: 每个阶段自动写入 `.super-dev/pipeline-state.json`，宿主可读取了解当前进度。
- **Pipeline 成本追踪**: 每阶段耗时/文件数自动记录到 `.super-dev/metrics/pipeline-cost.json`。
- **成本数据 finally 保护**: 即使 pipeline 异常中断，成本数据也会保存。
- **状态变更事件系统**: phase_started / phase_completed / phase_failed / pipeline_completed 自动触发 SessionBrief 更新和记忆提取。

### CLAUDE.md 知识文件引用

- 生成的 `.claude/CLAUDE.md` 现在自动包含 `@./knowledge/...` 引用。
- 宿主原生处理 知识引用指令，自动加载对应技术栈知识文件。
- 支持前端: Next.js / React / Vue / Nuxt / Angular / Svelte。
- 支持后端: FastAPI / Django / NestJS / Express / Spring Boot。
- 支持数据库: PostgreSQL / MySQL / MongoDB / Redis。

### 条件规则系统

- 新模块 `super_dev/rules/` — 支持 `.super-dev/rules/*.md` 条件规则。
- 规则可通过 frontmatter `paths: ["src/**/*.tsx"]` 指定只对特定文件生效。
- 支持排除模式 `!test/**`。

### 用户体验增强

- **首次使用引导**: 3 步快速开始面板，最多显示 4 次后自动隐藏。
- **Tips 提示系统**: 根据当前阶段显示上下文相关的操作建议。
- **项目模板**: `super-dev init --template ecommerce/saas/dashboard/mobile/api/blog/miniapp`。
- **`doctor --fix`**: 自动修复检测到的安装问题。
- **Shell 补全**: `super-dev completion bash/zsh/fish`。
- **版本更新检查**: PyPI 24h 缓存，有新版时提示升级。
- **`super-dev feedback`**: 快速打开 GitHub Issues 反馈。
- **`super-dev migrate`**: 2.2.0 → 2.3.0 一键迁移。

### 验证脚本增强
