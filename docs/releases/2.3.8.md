# v2.3.8 Release Notes

发布日期：2026-04-14

## 新增（Features）

### SEEAI 比赛验收升级为结构化证据

以前 SEEAI 链路的"验收通过"只靠宿主自觉回 `passed`，这一版起开始用结构化证据卡住"假通过"。

- 比赛证据现在按四段固定结构记录：`first_response`、`runtime_checkpoint`、`fallback_decision`、`demo_path`
- 通过条件收紧：SEEAI 宿主链路下，证据缺段或填写过浅时，即使 `status=passed`，`ready_for_delivery` 仍为 `false`，并给出 `SEEAI 比赛验收证据不完整 / 内容过浅` 的明确 blocker 与下一步命令
- 写入入口：
  - CLI：`super-dev integrate validate ... --competition-evidence-json '{...}'`
  - Web：`POST /api/hosts/runtime-validation` 请求体新增 `competition_evidence` 字段
- 输出回显：runtime validation 报告新增 `Evidence Ready` / `Missing Sections` / `Shallow Sections` 三行，宿主一眼就能看到缺哪段、哪段太水

### 设计灵感工作流（PR #11，感谢 @staruhub 超哥贡献）

UIUX 阶段引入 design inspiration prompt，把配色、版式、组件参考自动接入三文档与 Spec。本次升级由社区贡献者超哥（[@staruhub](https://github.com/staruhub)）通过 [PR #11](https://github.com/shangyankeji/super-dev/pull/11) 提交，感谢！

主要落点：

- `cli_experience_mixin.py` / `cli_parser_mixin.py` 暴露设计灵感子命令
- `design/ui_intelligence.py` 扩展灵感来源解析
- `creators/document_generator_content_mixin.py` / `prompt_templates.py` 把灵感片段写入 PRD 与 UIUX 文档
- 配置面新增灵感校验（`config/schema_validator.py`）

### 证据内容质量校验

仅有字段非空还不够，新加两道内容质量门：

- **最小内容长度**：每段聚合文本 ≥ 8 字符，封堵 `"ok"` / `"done"` 单字糊弄
- **模板关键词覆盖**：宿主提供 `competition_evidence_template.required` 时，段落必须双向 substring 命中至少一个关键词，封堵偏题填写
- 新增公开 API：`competition_evidence_shallow_sections(payload, template)` 与 `evaluate_competition_evidence(payload, template)`

## 修复（Fixes）

### WorkBuddy doctor 不再报 user_surfaces 缺失
- 从 `NO_SKILL_TARGETS` 移除 WorkBuddy
- `super-dev onboard --host workbuddy` 自动安装 Skill 到 `~/.workbuddy/skills/super-dev/` 和 `~/.workbuddy/skills/super-dev-seeai/`
- doctor 现在对 WorkBuddy 显示 "已就绪"

### 补充 Skill 不再重复安装报错
- `super-dev-seeai` 已存在时静默跳过，不需要 `--force`
- 解决 Gemini CLI、OpenCode、Windsurf、Trae 等宿主的 "Target skill already exists" 错误

### 全局清理 super-dev-core 残留
- `cleanup_all_legacy()` 扫描所有 `~/.**/skills/super-dev-core` 目录
- 支持未在官方列表里的宿主路径（`.iflow`、`.kimi`、`.qoderwork` 等）
- 每次运行 `super-dev` 时自动执行

### 其他修复
- WorkBuddy 文案更新为 "通过官方 Skills 接入"
- CI 配置重写为 Python CLI（原来是 Node.js 全栈模板，2 秒失败）
- README 媒体路径指向 `super-dev-website/public/`
- `.gitignore` 补全运行时状态路径

## 升级方式

```bash
pip install -U super-dev
# 或
super-dev update
```

升级后自动：
- 清理旧版本残留（`__editable__.super_dev-*.pth`、`super_dev-*.dist-info`）
- 清理所有宿主下的 `super-dev-core` 目录
- 刷新已安装的 Skills 到当前版本
- WorkBuddy 自动装上 Skill
