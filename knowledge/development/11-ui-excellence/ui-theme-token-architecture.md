## UI主题与Token架构

### 分层模型
- 基础Token：颜色、字号、圆角、间距、阴影。
- 语义Token：primary、success、warning、error、info。
- 组件Token：按钮、输入框、卡片、弹层专用映射。

### 主题策略
- 必须支持亮色、暗色、高对比三种主题。
- 主题切换不得出现闪烁与可读性下降。
- 多品牌场景使用品牌扩展Token，不破坏语义层。

### 工程约束
- 禁止直接写死品牌色与状态色。
- 主题变量必须可在构建时和运行时切换。
- 组件样式必须仅依赖语义Token。
