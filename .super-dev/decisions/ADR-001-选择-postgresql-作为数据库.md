# ADR-001: 选择 postgresql 作为数据库

**状态**: accepted
**日期**: 2026-03-29

## 背景 (Context)

项目需要选择数据库。经过对主流方案的评估，需要在以下方案中做出选择。评估维度包括：性能、生态、团队熟悉度、维护成本和长期演进。

## 考虑的方案 (Options)

### 方案 1: PostgreSQL (适配分: 85)

**优势**:
- ACID 事务
- 丰富的扩展生态
- JSON/JSONB 支持
- 成熟的社区

**劣势**:
- 水平扩展较复杂
- 配置调优门槛较高

### 方案 2: MySQL (适配分: 75)

**优势**:
- 广泛使用
- 生态成熟
- 读性能优秀

**劣势**:
- JSON 支持较弱
- 存储引擎碎片化

### 方案 3: MongoDB (适配分: 70)

**优势**:
- 灵活 Schema
- 水平扩展方便
- 文档模型直觉

**劣势**:
- 无原生 ACID（多文档）
- 存储开销较大

## 决策 (Decision)

选择 postgresql 作为项目的数据库。

## 理由 (Rationale)

postgresql 的主要优势包括: ACID 事务、丰富的扩展生态、JSON/JSONB 支持、成熟的社区。综合适配评分最高。

## 后果 (Consequences)

正面: ACID 事务、丰富的扩展生态、JSON/JSONB 支持、成熟的社区。
负面/风险: 水平扩展较复杂、配置调优门槛较高。
需要团队熟悉 postgresql 的最佳实践。
