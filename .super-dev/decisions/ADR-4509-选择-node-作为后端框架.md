# ADR-4509: 选择 node 作为后端框架

**状态**: accepted
**日期**: 2026-04-07

## 背景 (Context)

项目需要选择后端框架。经过对主流方案的评估，需要在以下方案中做出选择。评估维度包括：性能、生态、团队熟悉度、维护成本和长期演进。

## 考虑的方案 (Options)

### 方案 1: FastAPI (适配分: 85)

**优势**:
- 自动文档 (OpenAPI)
- 高性能 (async)
- 类型提示

**劣势**:
- 生态不如 Django 全面
- ORM 需第三方

### 方案 2: Django (适配分: 75)

**优势**:
- 全栈框架
- ORM/Admin 开箱即用
- 安全默认

**劣势**:
- 单体倾向
- 异步支持晚

### 方案 3: Express (Node.js) (适配分: 75)

**优势**:
- 极简灵活
- npm 生态
- 前后端同语言

**劣势**:
- 缺少约定
- 回调地狱（需 async/await）

## 决策 (Decision)

选择 node 作为项目的后端框架。

## 理由 (Rationale)

node 的主要优势包括: 极简灵活、npm 生态、前后端同语言。综合适配评分最高。

## 后果 (Consequences)

正面: 极简灵活、npm 生态、前后端同语言。
负面/风险: 缺少约定、回调地狱（需 async/await）。
需要团队熟悉 node 的最佳实践。
