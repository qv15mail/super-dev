# ADR-002: 选择 react 作为前端框架

**状态**: accepted
**日期**: 2026-03-29

## 背景 (Context)

项目需要选择前端框架。经过对主流方案的评估，需要在以下方案中做出选择。评估维度包括：性能、生态、团队熟悉度、维护成本和长期演进。

## 考虑的方案 (Options)

### 方案 1: React (适配分: 85)

**优势**:
- 巨大生态
- 灵活组合
- SSR/SSG 支持好 (Next.js)

**劣势**:
- 无官方全家桶
- 学习曲线中等

### 方案 2: Vue (适配分: 80)

**优势**:
- 渐进式
- 中文社区活跃
- 官方全家桶

**劣势**:
- TS 集成历史包袱
- 大型项目最佳实践较少

### 方案 3: Angular (适配分: 65)

**优势**:
- 企业级全家桶
- 强类型
- 依赖注入

**劣势**:
- 学习曲线陡
- 包体积偏大

## 决策 (Decision)

选择 react 作为项目的前端框架。

## 理由 (Rationale)

react 的主要优势包括: 巨大生态、灵活组合、SSR/SSG 支持好 (Next.js)。综合适配评分最高。

## 后果 (Consequences)

正面: 巨大生态、灵活组合、SSR/SSG 支持好 (Next.js)。
负面/风险: 无官方全家桶、学习曲线中等。
需要团队熟悉 react 的最佳实践。
